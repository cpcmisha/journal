# Principios de producto

## 1. Escribir debe ser sencillo

Abrir, escribir y cerrar debe ser el flujo principal.

## 2. La privacidad es un derecho

La privacidad nunca será una función Premium.

## 3. El usuario es dueño de sus datos

La información pertenece al usuario y permanece bajo su control.

## 4. El usuario siempre podrá irse

La aplicación permitirá exportar toda la información mediante formatos abiertos.

## 5. La escritura es más importante que el formato

Markdown estará disponible, pero no será obligatorio conocerlo.

## 6. La IA será opcional

Misha Journal deberá funcionar completamente sin inteligencia artificial.

## 7. Nextcloud es el hogar natural

La aplicación se integrará con la identidad, permisos, almacenamiento y experiencia de Nextcloud.

## 8. La simplicidad siempre gana

No se añadirán funciones únicamente para ampliar una lista de características.

## 9. Las funciones responderán a necesidades reales

Cada función deberá estar justificada mediante uso real, una historia de usuario o una necesidad técnica.

## 10. El proyecto debe durar

Las decisiones técnicas deberán priorizar mantenibilidad, migraciones seguras y formatos duraderos.
